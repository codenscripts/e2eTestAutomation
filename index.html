Project Overview

This project showcases a robust End-to-End (E2E) automation framework developed for a complex e-commerce platform. It's designed to validate critical user journeys and business processes by interacting with the User Interface (UI), Application Programming Interfaces (APIs), and directly with the Database (DB).

The framework's primary goal is to ensure the seamless integration and correct functioning of all application layers, mimicking real user scenarios from start to finish. By doing so, it significantly increases confidence in software releases and facilitates early detection of critical bugs.

Conceptual diagram of UI, API, and DB interactions.

Technologies Used

A comprehensive stack ensuring full-spectrum test coverage and modern CI/CD practices.

Java Maven Cucumber (BDD) Selenium WebDriver RestAssured (API) JDBC (DB) TestNG Docker (Containerization) Jenkins (CI/CD) Log4j2

Key Features & Methodologies

Full-Stack Validation

Orchestrates UI actions, API calls, and Database queries for holistic system verification, ensuring data consistency across all layers.

BDD-Driven Development

Leverages Cucumber's Gherkin syntax for human-readable, executable specifications, fostering collaboration between technical and business teams.

Maintainable UI Automation

Implements the Page Object Model (POM) with Selenium WebDriver, creating robust, reusable, and easily maintainable UI test scripts.

API & DB Integration

Utilizes RestAssured for efficient backend API validations and JDBC for direct database integrity checks, enhancing test reliability and speed.

Containerized Execution

Docker containerization ensures a consistent, isolated, and portable test environment, effectively eliminating "it works on my machine" issues.

Automated CI/CD Pipeline

Integrated with Jenkins for automated test execution on code changes, providing continuous feedback and accelerating the software delivery lifecycle.
